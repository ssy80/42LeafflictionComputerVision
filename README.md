# Leaffliction

*This project has been created as part of the 42 Advanced curriculum by ssian and axlee.*

## Description

Leaffliction is a machine learning project that classifies plant leaf diseases from images. Given a photo of an apple or grape leaf, the model identifies which disease (if any) is present across 8 classes.

The pipeline covers the full ML workflow:

* **Distribution** — visualise the class balance of the raw dataset, saved as charts to `distribution/before/` and `distribution/after/`.
* **Augmentation** — expand the dataset by applying 6 transformations per image (flip, rotate, skew, crop, distortion, contrast), growing each class from ~100 to ~700 images. Output saved to `dataset/augmented/`.
* **Transformation** — apply computer vision preprocessing (Gaussian blur, mask, ROI, pseudolandmarks, colour histogram) to extract meaningful leaf features.
* **Training** — train separate CNNs for Apple and Grape (TensorFlow/Keras) with early stopping and TensorBoard logging. Models saved to `models/Apple/splited/` and `models/Grape/splited/`.
* **Prediction** — classify a single leaf image and display the original alongside its masked transformation and predicted disease label.

### Dataset Classes

| Apple | Grape |
|---|---|
| Apple\_Black\_rot | Grape\_Black\_rot |
| Apple\_healthy | Grape\_Esca |
| Apple\_rust | Grape\_healthy |
| Apple\_scab | Grape\_spot |

## Instructions

### Setup

Place your dataset images into `dataset/raw/` following the folder structure above, then run:

```bash
source scripts/setup.sh
```

> On machines with an NVIDIA GPU, `setup.sh` automatically installs CUDA-enabled TensorFlow and configures the library paths. On CPU-only machines (e.g. school PCs) it installs plain TensorFlow.

To tear down the environment:

```bash
source scripts/remove.sh
```

### Order of Execution

```bash
source scripts/setup.sh                                              # 1. set up environment
make distribution                                                    # 2. analyse raw dataset balance → distribution/before/
make augment                                                         # 3. expand dataset → dataset/augmented/
make distribution-after                                              # 4. verify balance after augmentation → distribution/after/
make train                                                           # 5. train Apple and Grape models
make predict IMG="dataset/test/Apple/image (1).JPG"                 # 6. predict Apple
make predict IMG="dataset/test/Grape/image (1).JPG" MODEL=models/Grape/splited  # 6. predict Grape
```

### Makefile Commands

| Command | Description |
|---|---|
| `make distribution` | Analyse raw dataset → saves charts to `distribution/before/` |
| `make distribution-after` | Analyse augmented dataset → saves charts to `distribution/after/` |
| `make augment` | Augment all raw images → `dataset/augmented/` |
| `make train` | Train Apple and Grape models → `models/Apple/splited/` and `models/Grape/splited/` |
| `make predict` | Predict the class of a test image |
| `make predict-apple` | Predict using the Apple model |
| `make predict-grape` | Predict using the Grape model |
| `make test` | Run the full pytest suite (32 tests) |
| `make lint` | Run flake8 linter on `src/` |
| `make logs` | Launch TensorBoard at `http://localhost:6006` |
| `make clean` | Remove models, logs, augmented data, distribution charts, and caches |
| `make fclean` | `clean` + remove virtual environment |

### Distribution

Charts are saved as PNG files — no window opens:

```bash
make distribution        # before augmentation → distribution/before/Apple/ and /Grape/
make distribution-after  # after augmentation  → distribution/after/Apple/  and /Grape/
```

Expected result:
- **Before**: ~100 images per class
- **After**: ~700 images per class (7× increase, all classes balanced)

### Augmentation

```bash
make augment
```

Reads from `dataset/raw/`, writes originals + 6 augmented variants per image to `dataset/augmented/`. The raw folder is never modified.

### Training

```bash
make train
```

Trains separate models for Apple and Grape. Reads from `configs/train.yaml`:

```yaml
image_size: [256, 256]
batch_size: 16
epochs: 10
autotune: true
```

> On low-memory machines (e.g. school PCs), change to `image_size: [128, 128]`, `batch_size: 8`, and `autotune: false`.

After training, only two files are kept per model:

```
models/Apple/splited/leaf_model.keras
models/Apple/splited/class_names.csv
```

### Prediction

```bash
make predict IMG="dataset/test/Apple/image (1).JPG"
make predict IMG="dataset/test/Grape/image (1).JPG" MODEL=models/Grape/splited
```

Displays the original image alongside its masked transformation and prints the predicted class label. Requires a trained model in `models/`.

### Transformation

Apply preprocessing to a single image or a full directory:

```bash
# Single image
python3 ./src/Transformation.py -src "./dataset/raw/Apple/Apple_Black_rot/image (1).JPG"

# Directory
python3 ./src/Transformation.py -src "./dataset/raw/Apple/Apple_Black_rot" -dst ./dataset/augmented/Apple/Apple_Black_rot
```

### Tests

```bash
make test
```

Runs 32 tests across augmentation, distribution, transformation, split, and utils modules.
